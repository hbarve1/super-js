# Super.js 0.0.1

**Release Date:** December 15, 2023

This is the initial alpha release of Super.js, marking the beginning of our journey to create a strict, clean, and efficient superset of JavaScript with built-in type safety.

## 🎯 What's Included

### Initial Project Structure

- **Core Compiler Architecture**
  - Modular compiler design
  - Extensible parser and lexer
  - Type checker foundation
  - AST manipulation utilities

- **Basic Lexer and Parser Implementation**
  - Token recognition for all JavaScript constructs
  - AST generation for type annotations
  - Error recovery mechanisms
  - Source location tracking

- **Type Checker Foundation**
  - Basic type inference engine
  - Type compatibility checking
  - Error reporting system
  - Type annotation validation

- **CLI Tool Setup**
  - Command-line interface
  - Basic compilation commands
  - Help and version information
  - Configuration file support

- **Basic Documentation**
  - Getting started guide
  - Language syntax overview
  - CLI reference
  - Examples and tutorials

### Core Language Features

- **Basic Type Annotations**
  - Primitive type annotations (`string`, `number`, `boolean`)
  - Array type annotations (`string[]`, `number[]`)
  - Object type annotations (`{ name: string, age: number }`)
  - Function type annotations (`(param: string) => number`)

- **Simple Interface Definitions**
  - Interface declaration syntax
  - Property type checking
  - Optional and required properties
  - Interface extension

- **Function Type Checking**
  - Parameter type validation
  - Return type checking
  - Function overload support
  - Generic function types

- **Variable Declarations**
  - Type annotations for variables
  - Type inference for initializers
  - Const and let declarations
  - Type checking for assignments

- **Basic Error Reporting**
  - Type mismatch errors
  - Missing property errors
  - Function signature errors
  - Compilation error messages

## 🚀 Getting Started

### Installation

```bash
# Install from npm (alpha release)
npm install -g superjs@0.0.1

# Or clone and build from source
git clone https://github.com/super-js/super-js.git
cd super-js
npm install
npm run build
```

### Your First Super.js File

```javascript
// hello.sjs
interface User {
  name: string;
  age: number;
}

function greet(user: User): string {
  return `Hello, ${user.name}! You are ${user.age} years old.`;
}

const user: User = {
  name: "Alice",
  age: 30
};

const message = greet(user);
console.log(message);
```

### Compilation

```bash
# Compile a single file
superjs build --source hello.sjs

# Run the compiled JavaScript
node hello.js
```

## 🔧 Available Commands

### Basic Compilation

```bash
# Compile a file
superjs build --source file.sjs

# Compile with output directory
superjs build --source file.sjs --outDir dist

# Show help
superjs --help

# Show version
superjs --version
```

### Type Checking

```bash
# Type check without compilation
superjs check --source file.sjs

# Type check with detailed output
superjs check --source file.sjs --verbose
```

## 🐛 Known Issues

### Type System Limitations

- Limited type inference for complex expressions
- No support for conditional types
- Basic error messages without suggestions
- No type narrowing in control flow

### Compiler Limitations

- No source map generation
- Limited error recovery
- No incremental compilation
- Basic performance optimization

### Tooling Limitations

- No development server
- No watch mode
- No configuration file support
- Limited CLI options
