# Super.js 0.2.0

**Release Date:** January 15, 2024

This release brings significant enhancements to the type system, compiler features, and development tooling, making Super.js even more powerful for building robust applications.

## 🚀 New Features

### Type System Enhancements

- **Advanced Generic Constraints**
  - Support for complex generic constraints with multiple bounds
  - Conditional type constraints for more flexible type definitions
  - Improved type inference for constrained generics

- **Conditional Types Support**
  - Full implementation of conditional types (`T extends U ? X : Y`)
  - Nested conditional types for complex type logic
  - Type inference improvements for conditional expressions

- **Mapped Types Implementation**
  - Built-in mapped types (`Partial<T>`, `Required<T>`, `Readonly<T>`)
  - Custom mapped type creation with key remapping
  - Conditional mapped types for advanced transformations

- **Type Guards with User-Defined Predicates**
  - Enhanced type guard functions with custom predicates
  - Improved type narrowing in control flow
  - Better support for runtime type checking

### Compiler Features

- **Source Map Generation**
  - Full source map support for debugging
  - Inline and external source map options
  - Accurate mapping for type annotations and transformations

- **Incremental Compilation**
  - Faster rebuilds for large projects
  - Smart dependency tracking
  - Cache invalidation based on file changes

- **Parallel Compilation Support**
  - Multi-threaded compilation for improved performance
  - Configurable thread pool size
  - Memory-efficient parallel processing

- **Better Error Reporting with Context**
  - Enhanced error messages with code snippets
  - Type error explanations with suggestions
  - Improved location reporting for complex errors

## 🔄 Breaking Changes

### Generic Constraint Syntax
The generic constraint syntax has been updated for better consistency:

```javascript
// Old syntax (deprecated)
interface Container<T extends Object> { }

// New syntax
interface Container<T extends object> { }
```

### Decorator Implementation
Decorator implementation has been updated to match the latest proposal:

```javascript
// Old syntax (deprecated)
@decorator
class Example { }

// New syntax
@decorator()
class Example { }
```

### Utility Type Behaviors
Some utility types have been updated for better consistency:

```javascript
// Partial now properly handles readonly properties
type Partial<T> = {
  [P in keyof T]?: T[P];
};
```

## 📋 Migration Guide

### Upgrading from 0.1.0

1. **Update Generic Constraints**
   ```bash
   # Run the migration helper
   superjs migrate --from 0.1.0 --to 0.2.0
   ```

2. **Review Decorator Usage**
   - Update decorator calls to use the new syntax
   - Check for any custom decorator implementations

3. **Test Utility Type Usage**
   - Verify that utility types work as expected
   - Update any custom type definitions if needed

### New Features to Try

- **Advanced Type Guards**
  ```javascript
  function isUser(value: unknown): value is User {
  ```
