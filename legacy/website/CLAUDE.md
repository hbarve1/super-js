# website/ — Agent Context

This is the **SuperJS marketing and documentation website** built with Next.js (App Router), Tailwind CSS v4, and MDX. It is deployed to Cloudflare Pages.

**Read `AGENTS.md` first** — this Next.js version has breaking API changes from what most models were trained on. Check `node_modules/next/dist/docs/` before writing any Next.js code.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 16 (App Router) |
| Styling | Tailwind CSS v4 (`@tailwindcss/postcss`) |
| MDX rendering | `next-mdx-remote` |
| Code editor | `@monaco-editor/react` |
| Syntax highlighting | `shiki` |

| 3D | `@react-three/fiber` + `@react-three/drei` |

## How docs are served

All documentation pages live as MDX files in `content/docs/`. The routing is handled by `src/app/docs/[...slug]/page.tsx`, which calls `getDocBySlug(slug)` from `src/lib/docs.ts`.

`src/lib/docs.ts` reads from `content/docs/` using Node.js `fs`. It uses `gray-matter` to parse frontmatter. The slug is derived from the filename (strip `.mdx` / `.md` extension).

**URL mapping:** `content/docs/intro.mdx` → `/docs/intro`

Frontmatter fields used by the nav:
```yaml
---
title: "Getting Started"
sidebar_position: 1
description: "Optional description for SEO meta"
---
```

The `sidebar_position` field controls ordering in the nav sidebar (lower = higher in list). Pages without `sidebar_position` default to position 99.

## How to add a new doc page

1. Create `content/docs/your-page-name.mdx` with frontmatter:
   ```mdx
   ---
   title: "Your Page Title"
   sidebar_position: 5
   description: "One-line description for SEO"
   ---

   ## Introduction

   Content here...
   ```
2. The page is automatically available at `/docs/your-page-name` — no routing changes needed.
3. The sidebar nav is auto-generated from `getDocNavTree()`, so the new page appears automatically.
4. Test locally: `npm run dev` and navigate to `http://localhost:3000/docs/your-page-name`.

## What NOT to change

- **`next-env.d.ts`** — auto-generated by Next.js. Do not edit or delete.
- **`public/pagefind/`** — generated by `pagefind` post-build. Not checked in; regenerated on each production build.
- **`.vercel/`** — Cloudflare/Vercel adapter output. Not checked in.
- **`node_modules/`** — never commit this.
- **`tailwind.config.ts`** — Tailwind v4 config. If you need to add utilities, use CSS custom properties in `globals.css` instead of extending the config unless absolutely necessary.
- **`wrangler.toml`** — Cloudflare project config. Only the deploy team should change `project-name` and `account_id`.

## Key invariants

1. **All doc pages use MDX frontmatter** with at minimum `title`. Missing `title` silently breaks the sidebar nav.
2. **No client-side doc fetching.** `src/lib/docs.ts` uses `fs` directly — it is server-only. Do not import it in client components.
3. **`generateStaticParams` in `[...slug]/page.tsx` drives SSG.** If you add a doc page, it will be statically generated at build time automatically via `getAllDocSlugs()`.
4. **Search is post-build only.** The `pagefind` index is built after `next build` in the `postbuild` script. Search does not work in `npm run dev`.
