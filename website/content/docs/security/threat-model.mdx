# SuperJS Threat Model

This document enumerates the security surfaces of the SuperJS compiler toolchain and the threats against each surface.

Format: STRIDE (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege).

---

## Threat Matrix

| Surface | Threat | STRIDE | Mitigation | Owner Stage | Status |
|---|---|---|---|---|---|
| Parser (source input) | Pathological nesting (100k nested parens) causes stack overflow | DoS | Parser depth limit (configurable, default 500); `SJS-P099` emitted | Stage 1 | Planned |
| Lexer (source input) | Enormous identifier (1M chars) causes OOM | DoS | Lexer token size limit (64KB default) | Stage 1 | Planned |
| Lexer (source input) | Bidirectional Unicode codepoints that make code look different than it executes | Spoofing | Detect BiDi codepoints in identifiers and string literals; emit `SJS-W012` / `SJS-L011`; block in `--strict` mode | Stage 1 | Planned |
| LSP server | Unauthenticated stdin messages from another process | Spoofing / Tampering | LSP over stdio only (no TCP by default); OS process isolation provides authentication | Stage 3 | Planned |
| LSP server | Malformed JSON-RPC message causes crash | DoS | Message parsing wrapped in try/catch; malformed messages return parse-error response | Stage 3 | Planned |
| LSP server | Memory exhaustion via large workspace | DoS | `lsp.memoryBudgetMB` config field; exceeding budget emits `SJS-W010` and reduces cache aggressiveness | Stage 3 | Planned |
| Playground (browser sandbox) | User-submitted SJS code executes outside sandbox | Elevation of Privilege | Cloudflare Workers isolates each evaluation; no filesystem access; network calls blocked | Stage 3 | Planned |
| Playground | Exfiltration of other users' submitted code via timing attack | Information Disclosure | Per-request isolation in Workers; no shared state between evaluations | Stage 3 | Planned |
| npm publish (`--provenance`) | Package contents tampered in transit or by registry | Tampering | `--provenance` flag on publish links npm package to signed GitHub Actions run via Sigstore | Stage 5 | Planned |
| npm publish | Dependency confusion attack on `@superjs/*` namespace | Tampering | Register all `@superjs/` sub-package names on npm immediately (even stubs); scoped package prevents squatting | Stage 0 | In Progress |
| Crash logs | Full file paths and symbol names in crash reports leak user source structure | Information Disclosure | Crash logs ship basenames only; `--crash-full` is opt-in; documented in SECURITY.md | Stage 0 | Done |
| `superjs build` | Malicious `superjs.config.json` with path traversal in `outDir` or `paths` | Tampering | Config schema validation rejects non-string and absolute-path `outDir`; Stage 1 normalizes paths | Stage 1 | Planned |
| Supply chain | Malicious transitive dependency injected via npm | Tampering | Dependabot auto-PRs; lockfile committed; `npm audit` in CI | Stage 0 | Done |

---

## Out of Scope

The following are not treated as security vulnerabilities in SuperJS:

- Compiler crashes caused by intentionally malformed `.sjs` source (report as bugs, not CVEs)
- Performance issues that are not exploitable remotely (report as perf issues)
- Vulnerabilities in user-generated compiled output (responsibility of the application author)
