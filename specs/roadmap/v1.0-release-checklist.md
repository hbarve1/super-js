# v1.0 Release Checklist

Working tracker for the `superjs@1.0.0` GA, derived from
[`stage-6-stability-and-launch.md`](./stage-6-stability-and-launch.md) Exit Criteria.

Items are grouped by **who can do them**: code-doable (an agent can build), big-3
(large, multi-PR features), human/external (maintainer-only). Check off as shipped.

> Status as of 2026-06-18. Stages 3–5 code work largely landed (lint, LSP, stdlib,
> build plugins, `verify`, init templates, RELEASING.md). Remaining = below.
>
> **Workstream specs:** each item below has a detailed agent-ready spec in
> [`specs/roadmap/v1.0/`](./v1.0/README.md).

---

## A. Code-doable (an agent can build these)

### A1. Spec freeze + grammar
- [ ] Assemble `spec/language.md` (~150pg) from per-feature sections (B8); cross-reference to grammar productions + error codes; polish prose.
- [ ] Spec-freeze PR with `spec: freeze for 1.0` commit; further changes require maintainer "spec exception".
- [ ] `spec/grammar.ebnf` parses every fixture in `tests/fixtures/parser/`; CI gate enforces.

### A2. Error-code reference
- [ ] One `spec/error-codes/SJS-EXXX.md` page per emitted code (~80 anticipated).
- [ ] CI gate: for every code the compiler emits, a page file exists.

### A3. Docs site infrastructure (C3)
- [ ] Pick Astro Starlight or Docusaurus (decide week 1).
- [ ] Full-text search (Algolia DocSearch or local Pagefind/Lunr).
- [ ] Sidebar nav from file tree: spec, tour, migration, API, stdlib, errors (80+), CLI.
- [ ] Sitemap, RSS, dark mode, mobile-responsive, per-page edit-on-GitHub link.
- [ ] Lighthouse CI ≥ 90 on home + 10 sampled error pages.
- [ ] Broken-link checker on every PR.

### A4. Docs content
- [ ] **Language tour** — 20 lessons in `docs/tour/`, ≤5 min each, every lesson runnable in playground. Lesson 18 = "Serverless handler patterns" (T2), NOT decorators.
- [ ] **Migration guide** — `docs/migration/` in 3 parts (syntax, idiom, library); per-package notes for all 30 top wrappers.
- [ ] **API reference** — `superjs docgen` over stdlib + wrappers → `docs/api/`.
- [ ] **"Why SJS"** — `docs/why/`: vs TS / ReScript / Flow / Hegel / Civet / Imba / Elm / PureScript; honest "what SJS loses"; ReScript comparison table; `mvb-fastify` demo.
- [ ] **Compat matrix** — sortable HTML at `docs/compat/`, consuming each wrapper `STATUS.md`.
- [ ] **Perf benchmarks** — `docs/perf/`: cold compile vs tsc/esbuild, warm rebuild, LSP P99, per-phase numbers.

### A5. migrate from-prototype (C2)
- [ ] Complete `superjs migrate from-prototype`: rewrite `prototype/`-layout imports → `packages/compiler`-layout; flag prototype-only `.sjs` constructs; emit `MIGRATION_REPORT.md`.
- [ ] Test against this repo's actual `prototype/` dir; commit reference output.

### A6. Governance + security automation (file-level)
- [ ] `STABILITY.md` — stability tiers (stable / beta / unstable).
- [ ] `docs/deprecation.md` — deprecation policy (6-month alias rule).
- [ ] GitHub Discussions RFC template in `rfcs/` + `CONTRIBUTING.md`.
- [ ] Dependabot config; CodeQL workflow; `SECURITY.md` disclosure + 7-day SLA.
- [ ] Threat model (Stage 0) reviewed/updated with surfaces found during build.

### A7. CI matrix
- [ ] Promote Node 24 from nightly to blocking matrix (C7), alongside Node 20 + 22 LTS.

### A8. Example fixture (Stage 5 entry gap)
- [ ] `examples/mvb-fastify/` — minimum-viable-backend Fastify fixture (T3); green; used as launch "5-minute backend" + "Why SJS" demo.

## B. Big-3 (large features — scope each as its own multi-PR milestone)

- [ ] **Playground** — server-side, Workers sandbox + rate-limit, demo mode (T5). Tour lesson links depend on it. LARGE.
- [ ] **DAP debugger** — launch config + sum-type inspect formatters; may need `@superjs/runtime` inspect. LARGE.
- [ ] **30 `@superjs/types-*` wrappers** — backend-first wave-1 (T1); translator hits 70% typed-surface gate (B7); compat matrix MVP. LARGE.

## C. Human / external (maintainer-only — agent cannot do)

- [ ] ⚠️ **Rotate leaked npm token** (exposed in chat) — BEFORE any publish.
- [ ] ⚠️ **Tag `v0.1.1`** to republish ReDoS-fixed packages.
- [ ] **VS Code extension → Marketplace** (with co-publisher per R5).
- [ ] **Beta program** — select 3 of 5 teams; onboard; ≥1k LOC each; weekly check-ins in `docs/beta/`.
- [ ] **`dynamic` usage survey (C5)** — each team runs `superjs doctor --json` + count script; aggregate in `docs/beta/dynamic-usage-survey.md`; if >20% file `docs/known-issues/dynamic-rate-v1.md`.
- [ ] **Bug bash** — 1-week structured session; fix all `severity=blocker`.
- [ ] **Performance pass** — hit targets: cold 10k LOC ≤2s, warm ≤100ms, LSP idle ≤250MB, P99 ≤200ms. Slip 1 sprint if missed.
- [ ] **LSP memory audit** — heap snapshot at 100k LOC; fix/document retained allocs.
- [ ] **External security review (S7)** — book at Stage 3 start; report at `docs/security-review.md`; fix all critical/high before RC.
- [ ] **CodeQL + npm audit clean** (no high/critical).
- [ ] **RC cycle** — `1.0.0-rc.1` → `rc.2` → `rc.3`, ≥2 weeks apart; fix blockers only.
- [ ] **`superjs@1.0.0`** — tag + publish to npm with `--provenance`; GitHub Release + changelog; Sigstore attestation.
- [ ] **Launch artefacts** — blog post; HN / lobste.rs / r/typescript drafts; press kit (logo, screenshots, sample code, TS one-pager); Discord/Discussions ≥50 members.
- [ ] **Trademark check** — "SuperJS" in USPTO + EU; explicit non-claim in LICENSE/README.
- [ ] **Domain** — verify `superjs.dev` (or alternative).
- [ ] **On-call rotation** — maintainer + co-maintainers OR documented solo-path (R10); SLA per `RELEASING.md`.
- [ ] **Public roadmap update** — "entering RC, what we cut, path to GA."

## Buffer (R9)

If at end of Sprint 6.1 **two or more** of {spec freeze, beta onboarding, security
review} are behind → invoke explicit **+2-week buffer** (Sprint 6.3 → 4 weeks).
Announce as README banner + `docs/blog/...-stage-6-buffer-invoked.md`. No silent slip.

---

## Recommended order for code work

1. **A8** `mvb-fastify` example (small; unblocks tour + Why-SJS demos).
2. **A2** error-code pages + CI gate (mechanical; high coverage value).
3. **A3 + A4** docs site infra, then content (biggest code chunk; tour needs playground for full link-check, so author lessons but gate playground links behind B1).
4. **A5** migrate from-prototype, **A6** governance files, **A7** Node 24 matrix.
5. **A1** spec assembly + freeze (do last — freezes after everything else stable).
6. **B1–B3** big-3 — each a separate milestone, only after A-items land.
