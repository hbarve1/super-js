## What v1.0 Is

SuperJS 1.0 is a **production-ready compile-to-JavaScript language** for TypeScript-fluent Node.js backend developers who want type safety without TypeScript's complexity tax.

It is:
- A strict superset of JavaScript тАФ every valid `.js` file is valid `.sjs`, zero migration cliff
- A static type checker with a sound type system (no `any`, no silent unsoundness)
- A single CLI binary that lexes, parses, type-checks, and emits ES2022 + source maps
- Drop-in compatible with the top 30 npm packages on day one

It is **not** (at v1.0): a self-hosted compiler, a native/WASM/LLVM target, a React-first language, or a TypeScript superset. Those are v2.0+.

## Language

### Type System

SJS uses **bidirectional type inference** (Dunfield-Krishnaswami) with structural subtyping.

```sjs
const x: number = 42          // explicit annotation
const y = x + 1               // inferred: number
```

Types are **non-nullable by default**. `T?` is the only nullable form.

```sjs
function findUser(id: number): string? {
  if (id === 1) return "Alice"
  return null
}

const name = findUser(42)     // type: string?
const safe = name ?? "Guest"  // type: string  (narrowed after ??)
```

Gradual typing: unannotated code runs; `dynamic` replaces `any` with runtime checking.

```sjs
const data: dynamic = fetchFromAPI()   // runtime-checked, not a soundness hole
```

### Primitives and Operators

All ES5+ primitives. Type rules follow ECMA-262 exactly:

```sjs
const a: number = 1 + 2          // OK
const b: string = "hello" + 1    // OK: string coercion
const c: number = "hello" + 1    // SJS-E001: type mismatch
```

`++`/`--` only on `number` and `dynamic` operands. `typeof`, `instanceof`, `in` all type-safe.

### Functions

```sjs
function add(a: number, b: number): number { return a + b }

const greet = (name: string): string => `Hello, ${name}!`

// Optional and default params
function connect(host: string, port: number = 3000): void { ... }

// Rest params
function sum(...nums: number[]): number { return nums.reduce((a, b) => a + b, 0) }
```

Generators (`function*`, `yield`) and async functions (`async`/`await`) are fully typed.

### Classes and Interfaces

Structural interfaces тАФ implicit satisfaction, no `implements` boilerplate:

```sjs
interface Serializable {
  toJSON(): string
}

class User {
  constructor(readonly name: string, readonly age: number) {}
  toJSON(): string { return JSON.stringify({ name: this.name, age: this.age }) }
}

function save(s: Serializable): void { ... }
save(new User("Alice", 30))   // OK тАФ User satisfies Serializable structurally
```

Generics with invariant type parameters (declared-site variance `+T`/`-T` in v1.1):

```sjs
class Stack<T> {
  private items: T[] = []
  push(item: T): void { this.items.push(item) }
  pop(): T? { return this.items.pop() ?? null }
}
```

### Sum Types and Match

Algebraic variants with exhaustiveness checking:

```sjs
type Shape =
  | Circle(radius: number)
  | Rect(width: number, height: number)
  | Triangle(base: number, height: number)

function area(s: Shape): number {
  return match s {
    Circle(r)       => Math.PI * r * r,
    Rect(w, h)      => w * h,
    Triangle(b, h)  => 0.5 * b * h,
  }
}
```

`match` is exhaustive тАФ missing a branch is a compile error (`SJS-E009`).

`Result<T, E>` is a first-class sum type, not a library convention:

```sjs
type Result<T, E> = Ok(T) | Err(E)

function divide(a: number, b: number): Result<number, string> {
  if (b === 0) return Err("division by zero")
  return Ok(a / b)
}
```

### Null Safety

`T?` is the only nullable form. Non-nullable by default means the compiler proves `null` cannot escape without a check:

```sjs
function greet(name: string): void {
  console.log(name.toUpperCase())   // safe: name is string, not string?
}

const maybeUser: User? = getUser()
greet(maybeUser)           // SJS-E005: argument is string? not string
greet(maybeUser ?? "Guest") // OK: narrowed to string
```

Null narrowing works in `if`, `??`, optional chaining `?.`, and `match`.

### JSX

On by default, no config required:

```sjs
function Button(props: { label: string; onClick: () => void }): JSX.Element {
  return <button onClick={props.onClick}>{props.label}</button>
}
```

JSX factory configurable via `superjs.config.json` (`jsxFactory`, `jsxFragment`).

### Banned Constructs

| Banned | SJS Alternative |
|--------|-----------------|
| `any` | `dynamic` (runtime-checked) |
| `T extends U ? A : B` | Sum types |
| `{ [K in keyof T]: ... }` | Structural interfaces |
| Template literal types | тАФ |
| `infer`, `namespace`, TS `enum` | Explicit constructs |
| `A & B` intersection types | Interface composition |
| `!` non-null assertion | Narrowing / `match` |

## Compiler

**Entry point:** `packages/compiler/` тАФ plain JavaScript, no Babel dependency.

**Pipeline:**

```
.sjs source
  тЖТ Lexer        (hand-written; тЙе 500k tokens/sec; BiDi-spoof rejection)
  тЖТ Parser       (recursive-descent/Pratt; тЙе 200k AST-nodes/sec; error recovery)
  тЖТ Type Checker (bidirectional; тЙе 50k constraints/sec; cross-file inference)
  тЖТ Codegen      (ES2022 + source maps v3; тЙе 100k AST-nodes/sec)
  тЖТ .js + .js.map
```

**Performance targets (cold, single-thread):**
- 10k LOC cold compile тЙд 2 s
- Single-file warm rebuild тЙд 100 ms
- LSP idle memory тЙд 250 MB
- LSP hover/completion P99 тЙд 200 ms

**Determinism gate:** same inputs тЖТ byte-identical outputs. Enforced in CI.

**Incremental compilation:** file-dependency graph + `apiHash`/`docHash` per module; cache key = `(file SHA-256, compiler version, config-hash)`. mtime not used.

**Error recovery:** parser never throws; emits up to 50 errors per file / 500 per build. Diagnostics validate against `spec/diagnostics.schema.json`.

## Interop

**`.d.ts тЖТ .d.sjs` translator** тАФ converts TypeScript declaration files to SJS interface files:
- `any` тЖТ `dynamic` with explicit annotation
- Union types тЖТ sum types where possible
- `?` optional тЖТ `T?` nullable
- Typed-surface score: тЙе 70 % on 200-package smoke test

**Top 30 npm packages** with hand-curated `.d.sjs` wrappers (backend-first):

Wave 1 (v1.0): `express`, `fastify`, `hono`, `zod`, `valibot`, `prisma`, `drizzle-orm`, `kysely`, `pino`, `winston`, `@aws-sdk/client-s3`, `@aws-sdk/client-lambda`, `@aws-sdk/client-dynamodb`, `axios`, `node-fetch`, `ioredis`, `pg`, `mysql2`, `mongoose`, `dotenv`, `lodash`, `date-fns`, `uuid`, `bcrypt`, `jsonwebtoken`, `ws`, `socket.io`, `bull`, `vitest`, `jest`

## Standard Library

Four layers, all published as separate npm packages with `--provenance`:

### Layer 0 тАФ Compiler built-ins (no import needed)
- `Result<T, E>`, `Option<T>` тАФ first-class sum types
- `panic`, `assert`, `todo`, `unreachable` тАФ runtime forms
- Iterator protocol тАФ `Symbol.iterator` wired automatically

### Layer 1 тАФ `@superjs/std-core`
- `Option<T>` combinators: `map`, `flatMap`, `unwrap`, `unwrapOr`, `filter`
- `Result<T, E>` combinators: `map`, `mapErr`, `andThen`, `orElse`, `unwrap`
- `pipe` and `flow` тАФ type-safe function composition

### Layer 2 тАФ `@superjs/std-collections`
- `List<T>` тАФ immutable persistent list
- `Map<K, V>` тАФ typed key-value store
- `Set<T>` тАФ typed set
- `Queue<T>`, `Stack<T>` тАФ typed FIFO/LIFO
- All implement `Iterator<T>`

### Layer 3 тАФ `@superjs/std-async`
- `AsyncResult<T, E>` тАФ `Promise`-backed `Result`
- `retry`, `timeout`, `race` combinators
- `Channel<T>` тАФ typed async queue (CSP-lite)

All modules bootstrapped: Stage 1 compiler compiles them; Stage 4 verifies Stage-1 output == Stage-4 output on downstream fixtures.
