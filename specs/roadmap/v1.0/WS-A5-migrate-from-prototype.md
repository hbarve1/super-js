# WS-A5: `superjs migrate from-prototype`

**Branch:** `feature/v1.0-migrate-proto`  
**Effort:** medium  
**Deps:** none — start immediately  
**PR base:** `main`

## Objective

Complete the `superjs migrate from-prototype` CLI subcommand (currently stubbed).
It rewrites a `prototype/`-layout SJS codebase to the `packages/compiler`-layout,
flags prototype-only constructs, and emits `MIGRATION_REPORT.md`.

## Context

- CLI app: `superjs/apps/cli/src/main.ts` — find the `migrate` subcommand (currently stubbed or minimal)
- Prototype directory: `super-js/prototype/` — actual prototype-era code to test against
- Current compiler layout: `superjs/libs/compiler/`, `superjs/libs/parser/`, etc.
- Banned constructs list: `specs/language/007-banned-features.md`
- The `superjs migrate from-ts` subcommand (TS→SJS) is separate; this one is prototype→current

## What the command does

```bash
superjs migrate from-prototype [--dry-run] [--out <dir>] <source-dir>
```

Given a directory of `.sjs` files written for the old prototype layout:

1. **Rewrite import paths** — prototype used `prototype/src/foo` layout;
   current layout uses `packages/compiler/src/foo` or the published `@superjs/*` packages.

2. **Flag prototype-only constructs** — things that existed in the prototype but are
   not in the current compiler. Examples:
   - `import { parse } from 'superjs/parser'` → `import { parse } from '@superjs/compiler'`
   - `import { compile } from 'superjs'` → `import { compile } from '@superjs/compiler'`
   - Any construct the current compiler emits SJS-E* for (test-compile after rewrite)

3. **Emit `MIGRATION_REPORT.md`** — a Markdown file listing:
   - Files rewritten (with diff summary)
   - Constructs flagged (code, file, line)
   - Manual steps required

4. **`--dry-run`** — print what would change without writing files.

5. **`--out <dir>`** — write rewritten files to a different directory (default: in-place).

## Import path rewrite rules

```typescript
// Prototype-era imports → current imports
const REWRITES: Array<[RegExp, string]> = [
  [/from ['"]superjs\/parser['"]/g,    "from '@superjs/compiler'"],
  [/from ['"]superjs\/checker['"]/g,   "from '@superjs/compiler'"],
  [/from ['"]superjs\/codegen['"]/g,   "from '@superjs/compiler'"],
  [/from ['"]superjs\/ir['"]/g,        "from '@superjs/compiler'"],
  [/from ['"]superjs['"]/g,            "from '@superjs/compiler'"],
  [/from ['"]\.\.\/prototype\/src\//g, "from '../superjs/libs/compiler/src/"],
  // Add more as discovered by reading prototype/ dir
];
```

Read the actual `super-js/prototype/` directory to discover the real import patterns used there.

## Prototype-only construct detection

After rewriting imports, run `superjs check` on each file.
Any SJS-E* or SJS-P* diagnostic = a construct that needs manual intervention.
Collect into `MIGRATION_REPORT.md`.

## `MIGRATION_REPORT.md` format

```markdown
# Migration Report

Generated by `superjs migrate from-prototype`.

## Summary
- Files processed: N
- Files rewritten: M
- Manual interventions required: K

## Rewritten import paths
| File | Old import | New import |
|------|-----------|-----------|
| src/foo.sjs | `superjs/parser` | `@superjs/compiler` |

## Manual interventions required
| File | Line | Code | Message |
|------|------|------|---------|
| src/bar.sjs | 42 | SJS-E004 | `any` is not valid; use `dynamic` |

## Next steps
1. Review each manual intervention above
2. Run `superjs check` on the rewritten codebase
3. Fix any remaining SJS-E* errors
```

## Implementation steps

1. Find the `migrate` subcommand in `superjs/apps/cli/src/main.ts`.
   Look for `from-prototype` sub-subcommand or add it.

2. Read the actual `super-js/prototype/` directory to understand what's there:
   ```bash
   find super-js/prototype -name "*.sjs" | head -20
   head -30 super-js/prototype/src/main.sjs  # or similar entry point
   ```
   Discover real import patterns.

3. Implement `migrateFromPrototype(sourceDir, options)` in
   `superjs/apps/cli/src/lib/migrate.ts` (new file):
   - Walk `.sjs` files in sourceDir
   - Apply REWRITES regex substitutions
   - Run `compile()` on each rewritten file to detect remaining issues
   - Build MIGRATION_REPORT entries
   - Write rewritten files (or dry-run)
   - Write `MIGRATION_REPORT.md`

4. Wire into CLI.

5. Test: run against `super-js/prototype/` directory:
   ```bash
   node superjs/apps/cli/dist/main.js migrate from-prototype \
     --dry-run super-js/prototype/
   ```
   Verify no crash; verify MIGRATION_REPORT.md is reasonable.

6. Commit reference output:
   ```bash
   node superjs/apps/cli/dist/main.js migrate from-prototype \
     --out /tmp/prototype-migrated super-js/prototype/
   cp /tmp/prototype-migrated/MIGRATION_REPORT.md \
     super-js/specs/design/prototype-migration-reference-output.md
   ```

7. Add CLI tests in `superjs/apps/cli/src/cli.spec.ts`:
   - `migrate from-prototype --help` works
   - `migrate from-prototype --dry-run <dir>` exits 0
   - `migrate from-prototype <tmpdir>` rewrites files + emits MIGRATION_REPORT.md
