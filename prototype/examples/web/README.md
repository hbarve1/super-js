# Super.js Todo App ‚Äî Web Demo

A full-featured Todo application written in **Super.js** (`.sjs`) and compiled
to vanilla JavaScript. No framework, no build system other than `superjs`.

## What this demo shows

| Feature | Where |
|---|---|
| Type aliases (`Todo`, `FilterMode`, `AppState`) | `src/app.sjs:1-12` |
| Union types (`"all" \| "active" \| "completed"`) | `src/app.sjs:8` |
| Typed function signatures with return types | every function in `src/app.sjs` |
| Generic factory function (`createStorage<T>`) | `src/storage.sjs` |
| LocalStorage persistence with typed wrapper | `src/storage.sjs` + `src/app.sjs` |
| Pure immutable state updates | `addTodo`, `toggleTodo`, `deleteTodo` |
| Typed DOM event handlers | `bindEvents` in `src/app.sjs` |
| Utility types (`debounce`, `throttle`) | `src/utils.sjs` |

The compiled `dist/app.js` contains **zero type annotations** ‚Äî they are fully
erased by the Super.js compiler. The browser only ever sees plain JavaScript.

## Compile and run

```bash
# From the prototype/ directory
npx ts-node src/cli.ts build \
  --source examples/web/src/app.sjs \
  --outDir examples/web/dist

# Or compile all three source files:
npx ts-node src/cli.ts build --source examples/web/src/utils.sjs   --outDir examples/web/dist
npx ts-node src/cli.ts build --source examples/web/src/storage.sjs --outDir examples/web/dist
npx ts-node src/cli.ts build --source examples/web/src/app.sjs     --outDir examples/web/dist

# Then open the app in your browser:
open examples/web/index.html
# or
npx serve examples/web
```

## Type safety in action

Super.js catches type errors at compile time. Here is an example of code that
**will be rejected** by `superjs build`:

```js
// src/app.sjs ‚Äî intentional type error
const state: AppState = {
  todos: [],
  filter: "all",
  nextId: 1,
}

// ERROR: Argument of type '"invalid"' is not assignable to
//        parameter of type 'FilterMode'
setFilter(state, "invalid")
```

Compiler output:

```
error[SJS-E003]: Argument of type '"invalid"' is not assignable
  to parameter of type '"all" | "active" | "completed"'
 --> app.sjs:9:16
 = spec: https://tc39.es/ecma262/#sec-function-calls
```

The same safety applies to the `Todo` shape ‚Äî passing an object with a missing
`id: number` field, or assigning a `string` where a `number` is expected, is
caught before any JavaScript reaches the browser.

## App features

- Add tasks by typing and pressing Enter or clicking "+ Add"
- Toggle individual tasks complete/incomplete
- Delete any task with the trash button (appears on hover)
- Toggle all tasks at once with the checkbox on the left of the input
- Filter by All / Active / Completed
- Clear all completed tasks at once
- Tasks persist across page reloads via `localStorage`
- Keyboard: `Escape` clears the input field
- Responsive layout, accessible markup (ARIA attributes), dark mode support
